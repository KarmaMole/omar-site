# Dispatch "Generate All" Feature

**Date:** 2026-04-09
**Status:** Approved

## Overview

A single "Generate All" button in the Payload admin dispatch edit sidebar that sends the article body to Claude and fal.ai, populating five fields in one click: excerpt, tags, SEO description, cover image, and meta image.

## User Flow

1. User writes/pastes dispatch body, sets title
2. User clicks **"Generate All"** in the sidebar
3. Button shows spinner/progress state
4. Server-side API route orchestrates AI calls
5. Fields are populated: `excerpt`, `tags`, `meta.description`, `coverImage`, `meta.image` (same as cover)
6. Brief "Generated! Undo?" toast appears for ~5 seconds to revert all fields if needed

## API Route: `POST /api/generate-dispatch-meta`

### Authentication

Route checks for an authenticated Payload user before proceeding. Returns 401 otherwise.

### Input

```json
{ "title": "string", "body": "string" }
```

### Processing Steps

**Step 1 — Claude call (single request, Haiku for speed/cost):**

Sends title + body with a system prompt instructing Claude to return JSON:

```json
{
  "excerpt": "1-2 punchy sentences suitable for a card layout",
  "tags": "comma-separated tags matching existing dispatch tag style",
  "description": "SEO meta description, ~155 characters",
  "imagePrompt": "filled New Yorker template based on article content"
}
```

Claude fills in the bracketed sections of the image prompt template (see below) based on the article's themes, metaphors, and content.

**Step 2 — fal.ai Nano Banana Pro call:**

- Model: `fal-ai/nano-banana-pro`
- Resolution: `2K`
- Aspect ratio: `16:9` (hero image format)
- Output format: `png`
- Prompt: the `imagePrompt` from Step 1

**Step 3 — Media upload:**

Download the image from fal.ai's returned URL, then upload to Payload's media collection via the local API. This returns a media document ID.

### Response

```json
{
  "excerpt": "string",
  "tags": "string",
  "description": "string",
  "coverImageId": "string",
  "imagePrompt": "string"
}
```

## Image Prompt Template

```
A New Yorker-style black-and-white ink illustration. In the center of the frame, [main subject or scene] is shown [doing something visually clear]. The image should express [core metaphor, contradiction, or irony] in a quiet, understated way.

Composition: The entire composition is compact and self-contained in the middle of the frame, with generous empty negative space surrounding it. No important element should touch the edges or rise too high in the frame unless explicitly intended.

Lighting: Soft, flat editorial lighting with subtle cross-hatched shadows and restrained tonal contrast. The image should feel calm, observational, and slightly austere rather than dramatic.

Color accent: Include a single subtle cyan (#00d9ff) element as the only color in the image, placed deliberately to draw the eye and reinforce the central idea.

Style: Minimalist pen-and-ink editorial illustration with fine cross-hatching, delicate stippling, crisp linework, and visible paper grain; quiet, intelligent, slightly ironic tone reminiscent of classic New Yorker-style conceptual illustration.

Fidelity: Clean anatomy, accurate perspective, balanced spacing, clear silhouette hierarchy, controlled detail, and strong visual clarity.

Negative prompt: photographic realism, painterly rendering, extra figures, cluttered background, chaotic composition, distorted anatomy, warped objects, unreadable text, heavy shading, excessive detail, bright colors, dramatic action.
```

Claude replaces the three bracketed sections `[main subject or scene]`, `[doing something visually clear]`, and `[core metaphor, contradiction, or irony]` based on the article content.

## Admin Component

### Location

Custom Payload UI field in the sidebar of the dispatch (blog-posts) collection edit form.

### Component: `GenerateAllButton`

- Path: `@/components/admin/generate-all-button`
- Registered as a `ui` type field in BlogPosts collection with `admin.position: "sidebar"`

### States

| State | Display |
|-------|---------|
| Idle | "Generate All" button |
| Generating | Spinner + "Generating..." (disabled) |
| Done | "Generated! Undo?" toast for ~5 seconds |
| Error | Red error message with retry option |

### Undo Mechanism

Before making the API call, the component snapshots current values of all five fields. The "Undo?" link in the toast restores them. After 5 seconds or navigation, the snapshot is discarded.

### Field Population

The component uses Payload's `useForm` / `useField` hooks to programmatically set:
- `excerpt`
- `tags`
- `meta.title` (unchanged, already auto-generated by SEO plugin)
- `meta.description`
- `coverImage` (media doc ID)
- `meta.image` (same media doc ID)

## Security

- `ANTHROPIC_API_KEY` and `FAL_KEY` are only accessed server-side in the API route
- Never sent to the client
- Not prefixed with `NEXT_PUBLIC_`
- API route validates Payload authentication before processing
